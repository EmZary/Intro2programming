My Introduction to Programming Notes

How the Internet Works

The client can be either the computer, the browser, or other device you are using to interface with the internet. The internet interacts with servers. The servers relay information through the internet and then the information is recieved by the client.

Languages

There are different languages of programing that give the computer different sets of instructions.

  • HTTP or HyperText Transfer Protocols is the language that the internet and servers use to communicate.
  • HTML or HyperText Markup Language is the language most websites are written in
  • CSS or Casscading Style Sheet is the style or look of a webpage. CSS interacts with the HTML text
  • DOM or Document Object Model which is a tree like structure that both CSS and HTML use. While not a language is important to the languages as it is the structure they are written in .

With each Programing Language there are tags (for HTML) and selectors (for CSS) which are comands that tell how to interact with the content.

HTML Tags

  • <b> text </b> will bold the text inside
  • <em> text </em> will put the text in italics for empasis
  • <a herf= "url"> text </a> will creat a hyperlink for the text.
    A hyperlink is a redirection to another page or part of the page.
  • <img src= "url" alt= "text"> adds an image to your page or (if the image is unavailable) alternate text
  • <p> text </p> will enclose text in a paragraph
  • <br> adds white space
  • <div> content </div> creates a box element
  • <span> content </span> creates an in-line element
  • Adding class = "text" to a tag will add an ID to the tag.

With HTML tags there are two different types of elements box and in-line.
Box elements create a box around text and other elements. Some examples of box elements are <p> and <div>.
Inline elements are in the line of the text. Some examples of inline elements are <br> and <span>.

CSS

With CSS their are a few things we need to know before we move on to the tags. The order matters. In most cases we order things from the biggest element to the smallest but in CSS it orders from the smallest to the largest. An example of this is that the CSS could apply the style to a word, then a sentance, then a paragraph, then a page, then the document, ect. This is important to remember when you write your CSS. Some other things to know are how things are labeled in CSS.

  • Style can refer to HTML elements, attribute, the general look of a page, or it can be a verb
  • Rule is a line of CSS code
  • Property is what you want to change
  • Value attributtes a value to a property
  • Selector is the HTML element you are applying the style to
  • Class a group of elements
  • ID a unique identifier

When writing in CSS you write a rule set using this formula [ selector {property:value; property:value;}] each property:value; set is called a declaration

You can also add comments to your code for more information, ideas for later, or for other programers. To add a comment in HTML open it with <!-- then write your comment. Be sure to close your comment with -->. To add a comment in CSS open it with /* and close it with */. For more information on HTML and CSS tags and selectors click here for HTML and here for CSS

The Box Model

The Box Model is a way to think about layout in CSS.

  • ContentThe image, text, or other elements
  • PaddingThe space between the content and the botder. The color for the padding is the background color
  • BorderGoes around the padding and the content. The color of the boarder is defined by the color property
  • MarginThe space between boxes. The margin is transparent.

To use the box model in CSS use the rule {box-sizing: border-box;} <div> are blocks and by defult they take up the entire width of the page. By adding the rule {display;flex} lets <div> elements apear next to each other.

The Programming Way

It is easy to thinkt hat you can just start typing out the HTML and the CSS and make a completed project however there are steps to take to make it easier.

Code, Test, Refine

  1. Look for boxes: if you are handed text, a mock up, or just told what you need to create the first step is to look for boxes in your work. These boxes will be the basis for how you write bothe the HTML and the CSS.
  2. Look for repeated styles and elements: this will help sort the boxes into smaller and smaller boxes.
  3. Write your HTML: using the boxes and repeated style and elements to create clean clear code.
  4. Write you CSS: using the elements created in the HTML.
  5. Test: try out your code on different platforms.
  6. Fix: fix any issues in your code.
  7. Repeat: repeat steps 3 through 7 until your product looks how it should
